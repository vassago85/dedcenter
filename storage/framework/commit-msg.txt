Rename Executive Summary -> Full Match Report; switch post-match PDFs to tall digital page

Rename (user-facing text only; routes/filenames preserved for stability):
- Hub button "Executive Summary (PDF)" -> "Full Match Report (PDF)" on
  org and admin match hubs.
- PDF <title>, in-page subtitle band, and downloaded filename slug all
  updated to "Full Match Report".
- Controller: new pdfFullMatchReport() action; legacy pdfExecutiveSummary
  and pdfPostMatchReport both delegate to it so historical callers keep
  working.
- PostMatchReportDataTest assertion updated to match the new branding.

Page geometry (digital-first):
- Dark navy backgrounds are not print-friendly, so we stop pretending
  these PDFs are printable. Both the Full Match Report and Standings
  now render as one tall continuous page via @page size: 210mm auto.
- pdfExecutiveSummary no longer passes customSize -> Gotenberg honours
  the CSS @page rule (preferCssPageSize stays true). Previously the
  controller was still forcing landscape dimensions on the renderer,
  which would have fought our portrait @page rule.
- Width stays at 210mm so the typographic scale keeps matching the
  A4-wide shooter report; height is auto so there are no horizontal
  page breaks, no orphan rows, no thead-repeat juggling.

UX copy:
- "Printable leaderboard" and "Printable" subtitles on the Standings
  tiles replaced with a neutral "Leaderboard" -- the report isn't
  optimised for print any more.

PWA:
- sw.js cache bumped to deadcenter-v24.
